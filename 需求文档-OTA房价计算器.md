# OTA 房价计算器需求文档

## 1. 项目目标

开发一个轻量在线房价计算网页，用于酒店运营人员快速计算携程、美团、飞猪等 OTA 平台的价格路径。

页面主要解决两个问题：

1. 正推计算：已知后台录入卖价，计算外网展示价、平台佣金、预计到手收入。
2. 倒推计算：已知希望外网展示的价格，倒推出后台应该录入的卖价；或已知希望酒店收到的金额，倒推出需要设置的外网价格和后台卖价。

项目目标是部署为 GitHub 公开页面，优先采用纯前端实现，不依赖后端数据库。

## 2. 使用对象

主要用户是酒店运营人员、价格运营人员、代运营人员。

典型使用习惯：

- 过去通过 Excel 函数手动计算。
- 需要频繁切换平台、活动折扣、佣金比例。
- 需要在铺价前快速判断后台卖价是否能实现目标外网价。
- 需要在看到外网价格后快速估算酒店实际到手收入。

## 3. 核心场景

### 3.1 场景一：后台铺价前倒推卖价

运营人员希望某个平台外网展示价达到指定目标，例如 199 元。

但平台后台实际需要录入的是卖价，中间会经过活动折扣、促销折扣、平台规则等计算。因此运营人员需要从目标外网价倒推出后台应录入卖价。

输入：

- 平台：携程 / 美团 / 飞猪
- 目标外网展示价
- 活动折扣
- 佣金设置

输出：

- 建议后台录入卖价
- 预计外网展示价
- 预计平台佣金
- 预计酒店到手收入

### 3.2 场景二：已知外网价估算到手收入

运营人员看到某个平台外网展示价格后，希望知道这个价格大概能给酒店带来多少到手收入。

输入：

- 平台
- 外网展示价
- 平台佣金
- 活动折扣或活动后的价格口径

输出：

- 预计平台佣金金额
- 预计酒店到手收入

### 3.3 场景三：后台卖价正推外网价

运营人员已经知道后台准备录入的卖价，希望确认经过活动折扣后外网大概展示多少，以及酒店到手收入是否合理。

输入：

- 平台
- 后台录入卖价
- 活动折扣
- 佣金比例

输出：

- 预计外网展示价
- 预计平台佣金
- 预计到手收入

## 4. 平台范围

MVP 阶段支持三个平台：

- 携程
- 美团
- 飞猪

每个平台都应有独立配置，避免用户每次手动重复输入。

每个平台配置项：

- 平台名称
- 默认佣金比例
- 默认活动折扣
- 小数处理方式

说明：平台默认参数可以先存在浏览器本地，不需要账号系统。

## 5. 计算口径

### 5.1 基础字段

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| 后台卖价 | 运营人员在 OTA 后台录入的价格 | 238 |
| 活动折扣 | 平台活动、促销或运营设置的综合折扣 | 0.85 |
| 外网展示价 | 客人在 OTA 前台看到的价格 | 202.3 |
| 佣金比例 | 平台按成交价扣除的佣金比例 | 0.12 |
| 佣金金额 | 平台扣除金额 | 24.28 |
| 到手收入 | 扣除佣金后酒店预计收到的金额 | 178.02 |

说明：

- 活动折扣第一版只支持一个综合折扣，不拆分多层券、会员价、平台补贴。
- 默认展示四舍五入后的整数，不展示括号内精确小数。

### 5.2 默认正推公式

默认按“佣金影响到手收入，不直接影响外网展示价”的口径计算：

```text
外网展示价 = 后台卖价 × 活动折扣
佣金金额 = 外网展示价 × 佣金比例
到手收入 = 外网展示价 - 佣金金额
```

其中：

- 后台卖价是 OTA 后台录入的划线价或基础卖价。
- 外网展示价由后台卖价乘以运营人员设置的活动折扣得到。
- 佣金按外网展示价扣除。

### 5.3 默认倒推公式：目标外网价倒推后台卖价

```text
后台卖价 = 目标外网展示价 ÷ 活动折扣
佣金金额 = 目标外网展示价 × 佣金比例
到手收入 = 目标外网展示价 - 佣金金额
```

### 5.4 默认倒推公式：希望到手金额倒推价格

```text
外网展示价 = 希望到手金额 ÷ (1 - 佣金比例)
后台卖价 = 外网展示价 ÷ 活动折扣
```

### 5.5 待确认口径

部分平台或活动可能存在“卖价加佣”“佣金加价”“平台补贴”“满减券”等特殊规则。MVP 阶段先不复杂化，先提供一个基础口径：

1. 标准佣金模式：佣金只影响到手收入。
2. 卖价加佣模式：用户输入希望酒店拿到的金额，系统按佣金反推需要达到的外网价，再结合活动折扣反推后台录入卖价。

后续如果平台规则明确，再扩展为更细的计算链条。

已确认的 MVP 口径：

- 佣金 = 按外网展示价扣除。
- 活动折扣 = 后台卖价到外网展示价之间的综合折扣。
- 卖加佣金 = 为了让酒店收到指定金额，按佣金比例反推外网价，再反推后台卖价。
- 默认取整 = 四舍五入到整数，不展示两位小数。

## 6. 页面功能

### 6.1 首页主计算器

首页直接进入计算器，不做营销页。

主要区域：

- 平台选择：携程 / 美团 / 飞猪
- 计算模式选择：正推 / 倒推外网价 / 倒推到手收入
- 输入区域
- 结果区域

### 6.2 平台参数设置

用户可以为每个平台保存默认参数。

配置项：

- 佣金比例
- 活动折扣
- 小数取整方式

保存方式：

- 使用浏览器 localStorage 保存。
- 不需要登录。
- 不需要数据库。

### 6.3 结果展示

每次计算后显示：

- 后台应录入卖价
- 外网预计展示价
- 平台佣金金额
- 酒店预计到手收入
- 当前计算公式摘要

### 6.4 示例预设

为降低新用户理解成本，可以提供一组示例数据：

| 平台 | 后台卖价 | 活动折扣 | 佣金比例 |
| --- | ---: | ---: | ---: |
| 携程 | 238 | 0.85 | 0.12 |
| 美团 | 228 | 0.88 | 0.10 |
| 飞猪 | 218 | 0.90 | 0.08 |

示例仅用于演示，不代表真实平台规则。

## 7. 非功能要求

- 纯前端页面，便于 GitHub Pages 部署。
- 适配手机和电脑浏览器。
- 不依赖外部接口。
- 不保存敏感数据到服务器。
- 支持用户本地保存配置。
- 页面加载快，结构清晰。
- 用户输入数字、切换平台或切换模式时，结果应实时更新，不需要点击计算按钮。

## 8. 技术方案

MVP 推荐技术栈：

- HTML
- CSS
- JavaScript
- localStorage
- GitHub Pages

不建议 MVP 阶段引入：

- 后端服务
- 用户登录
- 数据库
- 复杂权限系统
- 平台自动抓取

项目文件建议：

```text
index.html
styles.css
app.js
README.md
```

## 9. MVP 范围

第一版必须完成：

- 三个平台切换。
- 每个平台可配置佣金和活动折扣。
- 支持正推计算。
- 支持目标外网价倒推后台卖价。
- 支持希望到手金额倒推后台卖价。
- 支持本地保存平台配置。
- 支持 GitHub Pages 静态部署。

第一版暂不做：

- 登录账号。
- 多酒店管理。
- 房型批量导入。
- Excel 导入导出。
- 平台真实 API 对接。
- 历史计算记录云端保存。
- 复杂优惠券叠加。

## 11. 开发顺序建议

1. 搭建静态页面结构。
2. 完成平台配置数据结构。
3. 完成三种计算模式。
4. 完成实时结果展示。
5. 接入 localStorage 保存配置。
6. 做移动端样式适配。
7. 编写 README 和 GitHub Pages 部署说明。

## 12. 需要后续确认的问题

- 是否需要保留历史计算记录。
- 是否需要支持一键复制计算结果。
- 是否需要支持 Excel 批量导入导出。
- 后续是否要为携程、美团、飞猪分别加入真实平台特殊规则。
